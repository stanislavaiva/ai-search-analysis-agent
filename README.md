# AI Search & Analysis Agent

LLM-агент с динамической маршрутизацией и state-driven pipeline,
поддерживающий multi-step обработку запросов 

Агент декомпозирует пользовательский запрос на последовательность интентов и выполняет их как pipeline.

Поддерживаемые задачи:

* поиск информации
* суммаризация
* анализ тональности
* прямые ответы без pipeline

---

## Архитектура

Агент реализован как state-driven execution pipeline.

### State (`AgentState`)

Единое состояние передаётся через все шаги:

| Поле                  | Описание                                 |
| --------------------- | ---------------------------------------- |
| `user_query`          | исходный запрос                          |
| `detected_intents`    | распознанные интенты                     |
| `execution_chain`     | нормализованная последовательность шагов |
| `intermediate_result` | результат текущего шага                  |
| `final_answer`        | итоговый ответ                           |
| `next_intent_index`   | текущая позиция в pipeline               |

---

### Execution Flow

```
intent_recognition  → handlers → finalizer
```

* **intent_recognition** — извлечение интентов из запроса
* **handlers** — выполнение шагов (`search`, `summarize`, `sentiment`)
* **finalizer** — формирование итогового ответа

---

### Поддерживаемые интенты

| Интент      | Назначение         |
| ----------- | ------------------ |
| `search`    | получение данных   |
| `summarize` | сжатие информации  |
| `sentiment` | оценка тональности |
| `none`      | прямой ответ       |

---

## Логика маршрутизации

Перед выполнением интенты нормализуются:

* удаляются дубликаты
* `none` удаляется, если есть другие интенты
* гарантируется порядок выполнения:

```
search → summarize → sentiment
```


## Обработка запросов без интента

Если интенты не распознаны или равны `none`, pipeline не используется.

Поведение:

1. Выполняется прямой вызов LLM
2. Ответ возвращается без дополнительных шагов

Покрывает:

* простые вопросы (`2+2`)
* диалоговые запросы
* некорректный ввод

---

## Обработка составных запросов

Для multi-step задач формируется цепочка шагов.

Пример:

```
Найди исследования и определи тональность
```

Pipeline:

```
search → summarize → sentiment
```

Передача данных между шагами:

| Шаг       | Вход          | Выход            |
| --------- | ------------- | ---------------- |
| search    | query         | найденные данные |
| summarize | search output | краткое резюме   |
| sentiment | summary       | оценка           |

## Работа с поиском

В качестве retrieval-слоя используется Tavily API.

### Pipeline поиска

Поиск состоит из двух этапов:

1. **Retrieval (Tavily API)**  
   Выполняется запрос к Tavily, который возвращает агрегированные результаты:

   - `title`
   - `url`
   - `snippet`

2. **Content Enrichment (HTML → text)**  
   Агент дополнительно загружает до 3 найденных источников:

   - выполняются HTTP-запросы к страницам
   - из HTML извлекается основной текст (`fetch_plain_text`)
   - удаляются служебные элементы (`script`, `style`)
   - текст нормализуется и обрезается

Полученные тексты объединяются с Tavily-результатами и передаются в LLM.

---

### Особенности

| Характеристика | Описание |
|---------------|---------|
| Тип поиска | retrieval + lightweight scraping |
| Глубина | top-3 источника |
| Парсинг | упрощённый (без DOM-анализа) |
| Назначение | улучшение качества контекста |

---

### Обработка ошибок

| Сценарий                  | Поведение                                       |
|--------------------------|-------------------------------------------------|
| Tavily не вернул результаты | используется fallback через LLM               |
| Ошибка загрузки страницы   | игнорируется, обработка продолжается           |
| Частично шумные данные     | сглаживаются на этапе суммаризации             |
## Failover-логика

Для каждого шага задаётся список моделей.

Алгоритм:

1. попытка выполнения на основной модели
2. при ошибке — переключение на следующую
3. при исчерпании списка — ошибка шага

Реализация:

```
iter_llms_for_node()
```

Покрывает:

* rate limits
* ошибки провайдеров
* нестабильные модели

## Работа с LLM

Взаимодействие инкапсулировано в `chains.py`.

Особенности:

* structured output 
* единый интерфейс вызова
* минимизация парсинга текста

### Case 1: поиск + пересказ

**Query:**

```
найди последние новости о ядерной энергетике и прескажи их на русском
```

**Output:**

```
Краткое резюме:
Ядерная энергетика является важной областью, где Россия лидирует в развитии передовой ядерной энергетики и является главным поставщиком ядерных технологий в мире. Компании, такие как "Ростех", разрабатывают новые поколения стойких к радиации камер для АЭС. Ядерные реакторы имеют различные применения, включая производство радиоизотопов, нейтронную визуализацию и изучение свойств материалов.

Ключевые факты:

1. Россия лидирует в развитии передовой ядерной энергетики и является главным поставщиком ядерных технологий в мире.
2. "Ростех" разрабатывает новые поколения стойких к радиации камер для АЭС.
3. Ядерные реакторы имеют различные применения, включая производство радиоизотопов, нейтронную визуализацию и изучение свойств материалов. 
```

---

### Case 2: поиск + анализ

**Query:**

```
Найди мнения о влиянии ИИ на рынок труда и определи их тональность
```

**Output:**

```
1. ИИ уже широко применяется в различных отраслях и изменяет рынок труда. -> нейтральная: констатация факта.
2. Это может привести к структурным изменениям в экономике в течение следующего десятилетия. -> нейтральная: прогноз без оценки.
3. ИИ создает новые возможности, но также требует от компаний и работников адаптации и развития новых навыков. -> смешанная: подчеркиваются как позитивные, так и негативные аспекты влияния ИИ.  
```

**Query:**

```
2+2
```

**Output:**

```
4
```

